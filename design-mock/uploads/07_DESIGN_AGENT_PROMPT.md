# Design Agent Prompt

You are designing a clickable React prototype for the next-generation GAD App / Workbench.

Important:

The existing GAD website ("Hearth") already exists and should NOT be redesigned from scratch.

The existing systems already solved:
- parsing
- replay rendering
- movement smoothing
- lerping
- encounter playback
- local LLM integration

Your task is NOT to replace those systems.

Your task is to redesign the EXPERIENCE around them.

# Core Goal

Transform the app from:
- replay-first
to:
- player-first.

The replay system becomes an advanced mode.

The primary experience should begin with:
- personal reflection,
- latest raid,
- support contribution,
- progression context,
- and local exploration.

# Key UX Goals

- player-centric onboarding
- progressive disclosure
- trustworthy local AI interactions
- calm contribution flow
- inspectable workflows
- low-noise visual design
- advanced replay preserved for deep users

# Critical Constraints

Do NOT design:
- esports dashboards
- engagement loops
- SaaS admin aesthetics
- noisy KPI walls
- mobile social feed patterns

Do design:
- local-first workshop feeling
- meaningful replay transitions
- calm raid review flows
- narrative clarity
- support contribution visibility
- trustworthy AI-assisted exploration

# Prototype Expectations

Produce:
- clickable React prototype
- realistic navigation
- fake/sample data
- multiple raid states
- replay entry points
- AI interaction examples
- contribution flow

The prototype should feel:
- technically sophisticated,
- emotionally calm,
- inspectable,
- and player-centered.
