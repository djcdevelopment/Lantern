# Existing Systems To Preserve

The new prototype should assume these systems already exist and work.

Do NOT redesign these systems from scratch.

## Existing Capabilities

Already implemented:
- combat log parsing
- replay rendering
- movement smoothing
- lerping
- encounter playback
- local LLM integration
- advanced raid review surfaces

The prototype should DESIGN AROUND these systems.

---

# Replay Philosophy

Replay is:
- important
- powerful
- technically solved

But replay should no longer be:
- the first emotional touchpoint.

Replay becomes:
- an advanced inspection layer.

---

# Existing Strengths

The existing replay UI already provides:
- team-level awareness
- movement review
- encounter visualization
- advanced analysis

The redesign should:
- improve onboarding
- improve readability
- improve player-first exploration
- improve emotional clarity
- improve contribution flows

NOT:
- throw away functioning replay systems.
